> **Scope:** Multi-chain (EVM, Solana, Sui/Move, Cosmos). Priority: DeFi AMMs, bridges, staking vaults.
> **Window:** Last ~48h (June 10–12 UTC); Gravity Bridge ($5.4M) included as highest-signal recent bridge incident.

## 1. 🔴 Active Exploits (last 24–48 h)

### 1.1 Raydium — $1.34M | Fake LP Token Validation Bypass | Solana
- **Date:** June 10–11, 2026
- **Chain:** Solana
- **Root cause:** Raydium's legacy AMM V3 program — formally "stopped" in 2021 but never closed on-chain — failed to validate the LP mint address when processing liquidity removal. The attacker created a counterfeit LP token mint and presented it to the contract, which accepted the fake tokens as legitimate pool ownership, enabling full drainage of 5 dormant pools.
- **Bug class:** `logic-error` (missing LP mint validation), `unverified-contract` (deprecated program ID still callable)
- **Amount lost:** ~$1.34M ($900K USDC + $357K SOL + $86K RAY)
- **Attack path:** Fake LP mint → liquidity removal against legacy AMM V3 program → bridged via deBridge to Ethereum → laundered via Tornado Cash
- **Response:** Raydium will repay from protocol treasury; legacy AMM V3 program IDs formally retired; full audit of all legacy code paths launched
- **Sources:**

  - [CryptoNews — Raydium Suffers $1.34M Exploit via Fake LP Tokens](https://cryptonews.com/news/raydium-exploit-fake-lp-tokens-deprecated-solana-pools/)
  - [CryptoTimes — GoPlus Reveals How Hacker Stole $1.34M (June 11)](https://www.cryptotimes.io/2026/06/11/raydium-exploit-update-goplus-reveals-how-hacker-stole-1-34m/)

### 1.2 Gravity Bridge (Ethereum ↔ Cosmos) — $5.4M | Validator Key Compromise | Ethereum / Cosmos
- **Date:** May 30, 2026 (flagged by PeckShield / Cyvers; bridge halted same day)
- **Chain:** Ethereum bridge contract → Cosmos
- **Root cause:** A signing key held by Gravity Bridge validators was compromised. The bridge's security model depends on a quorum of validator signatures; with enough keys forged or stolen, the system treats fake withdrawals as legitimate. The breach appears to sit at the authorization layer rather than in contract logic itself.
- **Bug class:** `key-management`, `bridge-dvn`
- **Stolen assets:** ~$4.3M USDC · $553K WETH · $434K USDT · $64K PAX Gold
- **Amount lost:** ~$5.4M
- **Status:** Bridge halted by validator set; investigation ongoing
- **Sources:**

### 1.3 Haedal Protocol Vault — $915K | Cross-Version Upgrade Logic Flaw | Sui
- **Date:** June 2026 (exact date not yet publicly confirmed)
- **Chain:** Sui
- **Root cause:** A 2025 upgrade to Haedal's Vault pool contracts introduced a logic divergence between old and new code paths. The deprecated *deposit* path was still reachable on-chain and minted LP shares using pre-upgrade math; the new *redemption* path calculated output using post-upgrade fair-value pricing. Attacker exploited the gap: deposit via old path at inflated share rate → redeem via new path → net excess underlying assets.
- **Bug class:** `upgradeability`, `logic-error`
- **Amount lost:** ~$915K
- **Response:** Contracts paused; full user compensation committed; patched upgrade in preparation
- **Source:** [SlowMist Hacked DB](https://hacked.slowmist.io/) · [crypto.news — Haedal suspends haeVault](https://crypto.news/sui-based-haedal-protocol-suspends-haevault-feature-following-cetus-223m-exploit/)

### 1.4 NovaBox — ~$107K | Flash Loan + Reward Distribution Race Condition | Ethereum
- **Date:** June 9, 2026
- **Chain:** Ethereum
- **Root cause:** Rewards were distributed to depositors *before* internal balances were updated — a classic check-effects-interactions (CEI) violation in the dividend accounting. Attacker borrowed 427.5 WETH via Aave V3 flash loan, deposited to inflate their share, collected dividends at an overstated ratio, then withdrew — draining 99.86% of the reward pool in a single transaction.
- **Bug class:** `flash-loan`, `logic-error`
- **Amount lost:** ~56.73 ETH (~$107K); 130+ depositors affected
- **Source:** [CryptoTimes — NovaBox Loses Nearly 99.86% ETH Reward Pool in Flash Loan Exploit](https://www.cryptotimes.io/2026/06/10/novabox-loses-nearly-99-86-eth-reward-pool-in-flash-loan-exploit/)

### 1.5 Ambient Finance — ~$110K | Flash Loan + Surplus Collateral Accounting Flaw | Ethereum
- **Date:** June 8, 2026
- **Chain:** Ethereum
- **Root cause:** An accounting logic flaw in Ambient Finance's surplus collateral handling allowed an attacker using a 50 ETH flash loan to cycle through proxy operations and net 83.72 ETH from the pool — a gain of ~33 ETH over the initial deposit. Ambient's single-contract DEX architecture means any accounting flaw affects the entire unified liquidity surface.
- **Bug class:** `flash-loan`, `logic-error`
- **Amount lost:** ~83.72 ETH (~$110K)
- **Source:** [CryptoTimes — Ethereum DeFi Protocol Ambient Finance Suffers $110K Drain](https://www.cryptotimes.io/2026/06/08/ethereum-defi-protocol-ambient-finance-suffers-110k-drain/)

## 2. 🟡 New Disclosed Findings / Techniques

### 2.1 Deprecated Code Path Exploitation — The Week's Dominant Meta-Pattern
- **Pattern:** Raydium and Haedal are independent exploits sharing one root meta-pattern: **dormant or deprecated code paths that were never fully decommissioned**. In Raydium, the AMM V3 program was "stopped" in 2021 but remained callable on-chain with shared pool accounts. In Haedal, a 2025 upgrade left old deposit entry points reachable alongside new redemption logic. In both cases the deprecated path lacked the validation discipline of the active path.
- **Where it applies:**
  - **Solana:** Old program IDs stay callable unless explicitly closed with `solana program close`. Programs that share PDAs or token accounts with newer versions are especially dangerous — the old program can still act on state the new program trusts.
  - **Sui/Move:** Module upgrades preserve old entry point signatures unless explicitly removed. Version guards must be applied at each entry point boundary.
  - **EVM (proxy patterns):** If a proxy's implementation is upgraded but the old implementation address still has `delegatecall` access or is accessible via a forwarding contract, the old logic remains live.
- **Sources:** [CCN — Raydium legacy pools](https://www.ccn.com/news/crypto/raydium-exploit-legacy-pools-solana/) · [SlowMist Hacked](https://hacked.slowmist.io/)

### 2.2 AI-Assisted Bytecode Decompilation of Unverified Contracts — Chainalysis Research (June 10)
- **Pattern:** Attackers are deploying AI-powered pipelines to reverse-engineer unverified smart contract bytecode, dramatically compressing the time from deployment to exploit. Confirmed 2026 victims include Truebit ($26.2M, integer overflow in Solidity v0.5.3 bonding curve), Aperture Finance, Ekubo, and Trusted Volumes. Cumulative H1 2026 losses from unverified contracts: **$36.7M**.
- **Where it applies:** Any protocol with unverified periphery contracts — helpers, fee collectors, proxy implementations — especially those compiled with Solidity <0.8.0.
- **Practical implication:** Unverified bytecode is no longer a security strategy; it's now a flag that accelerates attacker targeting.
- **Source:** [Chainalysis — Attackers Exploiting Unverified Smart Contracts](https://www.chainalysis.com/blog/attackers-exploiting-unverified-smart-contracts/)

### 2.3 Access Control Still #1 Vector by $ in 2026
- **Pattern:** Per the 2026 OWASP Smart Contract Top 10 dataset, access control failures account for **$953.2M** in losses vs $35.7M for reentrancy. The Humanity Protocol exploit (June 9, $32M) is a live example: a single compromised developer key held proxy admin + MINTER_ROLE, giving the attacker unchecked token minting authority over the entire supply.
- **Source:** [OWASP Smart Contract Security Top 10](https://scs.owasp.org/sctop10/) · [CoinDesk — Humanity Protocol hack](https://www.coindesk.com/tech/2026/06/09/humanity-protocol-token-crashes-more-than-80-after-a-usd32-million-private-key-hack)

## 4. 🎯 Hunt Hooks

**Hook 1 — Enumerate and attempt to call all legacy Solana program IDs on your targets.**
For any Solana protocol in scope, retrieve the full program deployment history. Check if older program IDs are still live (`solana program show <program_id>` → look for "not closed" status). If live, check whether they share token accounts or PDAs with the current program. Specifically look for: (a) `remove_liquidity` or `withdraw` instructions that accept user-supplied mint addresses without on-chain validation against an authority-controlled allowlist; (b) any instruction that trusts user-provided account addresses to be the "real" LP mint. This is exactly the Raydium pattern — low discovery cost, potentially critical finding on any multi-version Solana AMM.

**Hook 2 — Trace deposit and withdrawal code paths across upgrade versions in Sui/Move and EVM proxies.**
For any Sui/Move or EVM proxy-based protocol with upgrade history: map every entry point that can *create* a position/share, and separately map every entry point that can *redeem* a position/share. Check version guards on the deposit side. If a deprecated deposit entry point is still callable and computes shares using pre-upgrade math while the redemption path uses post-upgrade fair-value pricing, the spread between the two calculations is extractable value — this is the Haedal template. On EVM: also check that `delegatecall` forwarders don't route to old implementation addresses.

**Hook 3 — Audit CEI order in any custom dividend, staking-rewards, or yield-vault contract.**
In staking, dividend-pool, or yield-bearing vault contracts: search for functions where reward state is read/transferred before balance state is updated. Run: `grep -rn "transfer\|safeTransfer\|_distribute\|claimReward\|harvest" --include="*.sol"` and for each hit verify that `_balances[msg.sender]` (or equivalent) is updated *before* any external call or reward transfer. If rewards flow before balance updates, a flash-loan attacker can massively inflate their apparent share, collect outsized rewards, and repay — exactly as in NovaBox. This pattern recurs quarterly and is consistently underpaid in audits relative to its exploitability.
