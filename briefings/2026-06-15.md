### 1.1 Flooring Protocol — ~$500K Blue-Chip NFTs at Risk | BT404 Phantom Balance | Ethereum *(June 8, 2026 — Catch-Up)*

- **Chain:** Ethereum
- **Root cause:** A packed-storage + aggressive bit-manipulation pattern used in the protocol's **BT404-style** fractionalized NFT contract — originally written to minimize gas costs — introduced an **integer underflow in the fpToken (Fractional Pool Token) minting path.** By depositing a tiny amount of WETH, an attacker could trigger the underflow and receive a near-infinite fpToken balance (a "phantom ownership" state). That inflated balance was then redeemed against real NFT liquidity pools, draining them.
- **Kill chain:**
  1. Attacker calls the BT404 deposit function with 1 wei of WETH.
  2. Internal packed-storage arithmetic underflows → attacker's fpToken balance set to `uint256.max - 1` (phantom balance).
  3. Attacker calls `redeemNFT()` with the phantom balance → protocol releases locked NFTs from every affected pool.
  4. Blue-chip assets (BAYC, Mutant Apes, CryptoPunks, Azuki, Moonbirds) start leaving Flooring pools.
- **Whitehat rescue:** Yuga Labs VP of Blockchain (0xQuit), security researcher Coffee, and liquidity desk GrailsOTC detected the exploit in near-real-time and **used the same vulnerability defensively** to pull 68 high-value NFTs out of the draining pools before the attacker could claim them. 29 BAYC, 4 MAYC, 2 CryptoPunks, 1 Azuki, 2 Elementals, 26 Captains, 1 Moonbird, 2 Doodles rescued. Estimated value: ~$500K.
- **Affected scope:** The flaw lived in the shared BT404 contract model, meaning **both FloorProtocol V2 and BitmapPunks** pools were exposed, not just a single isolated market.
- **Status:** Protocol paused; asset recovery complete; full post-mortem expected.
- **Bug class:** `integer-overflow` (underflow), `rounding`
- **Amount at risk:** ~$500K (rescued via whitehat; no net loss to users)

## 2. 🟡 New Disclosed Findings / Techniques

### 2.1 AI Trading Agent Memory Poisoning — Emerging $45M+ Attack Vector in DeFi *(OWASP Advisory Published June 11, 2026)*

- **Pattern:** Autonomous AI trading agents — autonomous programs that manage portfolios, execute swaps, and control protocol parameters 24/7 — introduce a **new, non-Solidity attack surface: long-term memory stores (vector databases, RLHF fine-tuning datasets).** Two techniques in active use:
  1. **Memory poisoning:** Attacker injects malicious instructions into the agent's persistent memory (e.g., vector DB entries that look like trusted historical trade data). The payload sits dormant until a market trigger activates it, causing the agent to execute unauthorized transfers or privileged calls on-chain.
  2. **Prompt injection (session-scoped):** Malicious data arrives via external inputs the agent reads during a session — price feed updates, governance proposal bodies, on-chain memo fields, or even NFT metadata. The injected text redirects the agent's decision-making for that session.
- **Why DeFi-specific:** AI agents with on-chain signing authority can execute swaps, claim yields, vote in governance, or upgrade proxy contracts. A successfully poisoned agent is functionally equivalent to a compromised private key — but the attacker never needed to steal a key.
- **2026 impact:** $45M+ in direct losses across confirmed AI agent incidents. OWASP June 11 report confirms prompt injection drives the majority of production agentic AI failures observed.
- **Bug class:** `logic-error` (agent acts on attacker-controlled instructions), `key-management` (signing authority delegated to a corruptible system)
- **Applies to:** Any DeFi protocol using AI-powered vault managers, keeper bots with vector-database memory, LLM-based governance summarizers with execution authority, or on-chain AI agent frameworks.

### 2.2 BT404 / ERC-404 Derivatives — Shared Contract Model Risk

- **Pattern generalization:** The Flooring Protocol incident above confirms a known-but-underweighted risk: protocols that fork or extend **ERC-404 / BT404-style hybrid fungible+NFT contracts** often carry forward the original's gas-optimization bit tricks without re-auditing the downstream arithmetic. Because BT404-style contracts track fractional ownership using packed storage slots and bit-shifting (not standard `SafeMath`-protected fields), any rounding or underflow in the packing logic can flip a legitimate deposit into an astronomically large balance.
- **Applies to:** All fractional NFT protocols using BT404, DN404, or custom ERC-404 derivatives on Ethereum — especially those with multi-pool architectures where a single shared contract model serves multiple NFT collections (the blast radius expands proportionally to the number of pools).
- **Hunt signal:** Look for `assembly` blocks or `unchecked {}` in functions that set or update token balances, combined with bit-shift operations (`>> 128`, `<< 64`, etc.) used to pack two values into one `uint256`. A rounding edge case that results in a larger-than-expected balance in such a block is a critical finding.

## 4. 🎯 Hunt Hooks

**Hook 1 — Audit BT404 / ERC-404 hybrid contracts for packed-storage underflows.**
For any fractional NFT or hybrid fungible+NFT protocol in your scope: search for balance-setting logic that uses `assembly`, `unchecked {}`, or explicit bit-shift patterns (`>> 128`, `& 0xFFFF...`) instead of `+=` / `-=` with SafeMath. Specifically: (a) find the deposit-to-fpToken conversion function and trace what happens when the deposit amount is `1 wei` or any value that causes integer division to round down; (b) confirm that the minted balance cannot exceed the deposited value; (c) if multiple NFT collections share the same BT404 contract model (as in FloorProtocol V2 + BitmapPunks), a single underflow exploitable in one pool bleeds into all pools. A phantom-balance finding on a shared contract with >5 NFT pools is a critical-tier submission on any platform.

**Hook 2 — Check AI agent signing authority scope and memory integrity.**
If a protocol in your scope uses any AI keeper, AI vault manager, or LLM-driven governance summarizer that holds or can request on-chain signing authority: (a) determine whether the agent's long-term memory (vector DB, fine-tune dataset, retrieval cache) can be written to by untrusted data — price feed payloads, on-chain memos, NFT metadata, governance proposal text; (b) check whether there is a human-approval gate between the agent's decision and the actual on-chain execution — if the agent can sign and broadcast directly, memory poisoning = key compromise with no recovery path; (c) report any path where external on-chain data the agent reads during routine operation could contain executable-style instructions redirecting the agent's behavior. This attack class is brand-new to most bug bounty scopes — the absence of a rule doesn't mean it's out of scope; use OWASP LLM Top 10 as your framework reference.

**Hook 3 — Multisig time-lock gap hunting (Unleash Protocol / Humanity Protocol pattern).**
June 2026's dominant attack pattern: compromised administrative multisig with no meaningful time-lock. For every protocol where you can read the proxy admin address: (a) check whether the proxy admin is a multisig and, if so, whether there's a time-lock contract wrapping it (a raw `GnosisSafe` without a `TimelockController` is a red flag); (b) enumerate the signers — if any signer address is a hot wallet (externally owned account that signs transactions frequently), it is a high-probability target for the DPRK/Lazarus phishing pattern that took down Humanity Protocol this week; (c) verify that the multisig cannot in a single transaction both upgrade the implementation and drain the treasury — atomic upgrade+drain is the kill chain. This is currently the fastest path to a critical-tier access-control finding across the DeFi landscape.
